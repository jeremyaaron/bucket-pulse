Built on API Gateway, Lambda, DynamoDB, S3 Metadata Tables/Athena, and CloudFront. Auth via Cognito Hosted UI (PKCE) or external IdP.

Deploy in one command

CDK provisions infra, generates UI runtime config (env.json), builds Angular, and uploads to S3/CloudFront.

Inventory + Journal aware

Uses S3 Metadata Tables for inventory and journal metrics to compute freshness, staleness, anomalies, and alerts per prefix.

What you can do

Buckets List

  • Register buckets (prechecks metadata tables), search/filter, and see KPIs.
  • Click through to bucket details and prefix health.

Bucket Detail

  • View status, KPIs, recent alerts.
  • Manage tracked prefixes via modal with validation.

Prefix Health

  • Status pill/reason, freshness & staleness thresholds, recent alerts.
  • Evaluation history timeline with pagination; edit prefix thresholds.

Alerts

  • Filter by severity/type/bucket/prefix/date; paginate results.
  • Severity pills, resolved flag, links back to prefix health.

Explorer

  • Filter inventory by bucket/prefix, size/age ranges, storage class, and tags.
  • See object summaries and open details modal (metadata, tags, copy S3 URI/ETag).

How it works

  • Aggregation Lambda queries S3 Metadata Tables (inventory + journal) to compute prefix health.
  • Alerts stored in DynamoDB; optional SNS topic for fanout.
  • UI loads runtime config from env.json (API base, auth domain, client id, redirect).
