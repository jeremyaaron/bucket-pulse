See everything S3 is doing, from buckets to individual prefixes.

BucketPulse turns S3 Metadata Tables into a complete view of your S3 landscape—high-level bucket health, per-prefix behavior, alerts, and inventory exploration, all in one AWS-native UI.

Explore the UI View on GitHub

Buckets overview

See every monitored bucket at a glance

See every monitored bucket in one place with health status, region, tracked prefixes, and last evaluation time. Color-coded pills surface buckets with issues at a glance so you know where to drill in next.

  • Register buckets and run metadata prechecks.
  • Track KPIs like total objects, total bytes, and alerts in the last 24 hours.
  • Click through to bucket and prefix details in a single step.

Bucket detail

A cockpit for each critical bucket

Drill into a single bucket to see overall health, active alerts, and a list of all tracked prefixes with their freshness and staleness status. Understand the behavior of a bucket as a whole, not just a flat list of objects.

  • See aggregated health across all tracked prefixes in the bucket.
  • Inspect core metrics: total objects, total bytes, old data 90+ days, and last event time.
  • Review inline alerts and jump straight to prefix health views for deeper analysis.

Prefix health

Per-prefix heartbeat for your data flows

For critical prefixes, BucketPulse shows freshness intervals, staleness thresholds, event history, and age distributions so you know exactly how the data is behaving—and when something goes off the rails.

  • Define freshness thresholds (expected, warning, critical) for key prefixes.
  • Track staleness as the percentage of objects older than your chosen thresholds.
  • Review recent alerts with the underlying metrics that triggered each condition.

Alert feed

One alert stream for S3 anomalies

Instead of grepping logs or skimming Athena outputs, browse all freshness and staleness alerts in a single table with powerful filters. Perfect for on-call, incident reviews, and weekly reliability check-ins.

  • Filter by severity, alert type, bucket, prefix, and time range.
  • Expand any alert to see the raw metrics, thresholds, and time window involved.
  • Deep link back to the relevant bucket or prefix health views for root-cause analysis.

Inventory explorer

Answer “what’s in this prefix?” without writing SQL

Use the Explorer to filter objects by size, age, storage class, and tags on top of S3 Metadata Tables. Answer ad-hoc questions about what’s actually in a bucket or prefix without opening a query editor.

  • Start with bucket and prefix filters to scope your search.
  • Narrow results by size and age ranges, storage class, and tag key/value pairs.
  • Browse results in a paginated table, ready to feed into your next workflow or investigation.
